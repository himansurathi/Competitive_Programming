    Here is my basic algo:

    1. Break the number into 2 halfs "2133" = "21","33"
    2. Now if first half is greater than 2nd half AND first half is same
    size as 2nd half AND reverse(first) is greater than 2nd half, then
    nextPalindrome = first + reverse(first), otherwise go to step 3.
    Example 783322 = 783 + 387 = 783387. However 2133 wont hold. Nor will
    1001, as first = 10, and last = 01 = 1
    3. Increment the first half by 1,
    first = first+1
    nextPalindrome = first + reverse(first)
    Example 2133, first = 21, first+1 = 22, Now concat first and reverse first = 2222

                                            Kailash • 2 years ago

                                            Steps:
                                            1. Add 1 to the given no.(as we want next palindrome, greater then current number)
                                            2. keep i and j pointers on leftmost and rightmost digit respectively.
                                            3. now move both i and j inside i.e. towards middle digit.
                                            4. make digit at [j] equal to digit at [i] by adding something. if there is any carry from the above addition propagate towards MSB digit.
                                            5. now move j to next digit from right and follow step 4.
                                            6. repeat step 4,5 till i,j are on the middle digit/s.

                                            In this algorithm there is no need to handle any special cases.
